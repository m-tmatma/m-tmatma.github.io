OpenSSLをWindowsでビルド(32bit/64bit)

OpenSSL のコンパイル

OpenSSLを解凍したディレクトリを以下 c:\...\openssl-1.0.0m とします。
  1. 以下のコマンドを入力する

    OpenSSLでアセンブラを使わずにコンパイルする場合 

    c:\...\openssl-1.0.0m> CALL "C:\Program Files\Microsoft SDKs\Windows\v7.1\Bin\SetEnv.cmd" /x86 /release
c:\...\openssl-1.0.0m> perl Configure VC-WIN32
c:\...\openssl-1.0.0m> ms\do_ms
c:\...\openssl-1.0.0m> nmake -f ms\ntdll.mak
c:\...\openssl-1.0.0m> nmake -f ms\ntdll.mak install
    OpenSSLでMASM を使う場合(詳細は OpenSSL の INSTALL.W32 を読むこと) 
    c:\...\openssl-1.0.0m> CALL "C:\Program Files\Microsoft SDKs\Windows\v7.1\Bin\SetEnv.cmd" /x86 /release
c:\...\openssl-1.0.0m> perl Configure VC-WIN32
c:\...\openssl-1.0.0m> ms\do_masm
c:\...\openssl-1.0.0m> nmake -f ms\ntdll.mak
c:\...\openssl-1.0.0m> nmake -f ms\ntdll.mak install
    OpenSSLで x64 を使う場合(詳細は OpenSSL の INSTALL.W32 を読むこと) 
    c:\...\openssl-1.0.0m> CALL "C:\Program Files\Microsoft SDKs\Windows\v7.1\Bin\SetEnv.cmd" /x64 /release
c:\...\openssl-1.0.0m> perl Configure VC-WIN64A
c:\...\openssl-1.0.0m> ms\do_win64a
c:\...\openssl-1.0.0m> nmake -f ms\ntdll.mak
c:\...\openssl-1.0.0m> nmake -f ms\ntdll.mak install
